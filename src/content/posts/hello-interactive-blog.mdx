This blog is written in **MDX**: Markdown for the prose, with the option to drop
real React components right into the page. That means a post can be a plain essay
— or a tiny interactive playground — without changing any infrastructure.

## Markdown handles the basics

Everything you expect from Markdown works: **bold**, _italic_, `inline code`,
[links](/blog/git-from-the-inside-out/), lists, quotes, and tables.

- Images live in `public/` and are referenced by path
- Videos are a raw `<video>` tag or an embed
- Diagrams can be exported SVGs or generated from code

> The platform is boring on purpose. The energy goes into writing, not plumbing.

## Code blocks are syntax-highlighted

```ts
function gitHashObject(type: string, content: Uint8Array): string {
  const header = `${type} ${content.length}\0`;
  return sha1(concat(utf8(header), content)); // matches real git
}
```
